NTM (Named Tmux Manager) is a Go-based CLI tool that enables spawning, tiling, and coordinating multiple AI coding agents (Claude, Codex, Gemini) across tmux panes with an interactive TUI command palette. It's designed to orchestrate sophisticated multi-agent workflows where different AI instances work in parallel on complementary tasks.

## Key Features

- **Multi-agent coordination** — spawn and manage Claude, Codex, and Gemini agents simultaneously
- **Tmux integration** — automatically tile agents across tmux panes with intelligent window management
- **TUI command palette** — interactive interface for controlling agent execution and workflow direction
- **Swarm execution** — coordinate multiple AI instances on different tasks (e.g., implementing security recommendations in parallel)
- **Real-time feedback** — see all agents working in tiled view with clear output separation

## Use Cases

- **Security audit workflows** — spawn agents to implement security recommendations from frameworks like `security-audit-for-saas`
- **Parallel development** — assign different features/modules to different agents for concurrent implementation
- **Multi-perspective problem solving** — have different models tackle the same problem and compare approaches
- **Coordinated code generation** — generate and integrate code across multiple files in parallel
